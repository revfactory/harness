# 컨텍스트 작성 템플릿

> 이 문서는 사용자가 프로젝트 루트의 `context/` 파일과 루트 문서들을 작성할 때 참고하는 예시/가이드입니다.
> "팀 구성 템플릿"이 아닙니다. 사용자가 자신의 컨텍스트를 작성할 때 따라갈 수 있는 템플릿입니다.

---

## WORKING-RULES 섹션

```markdown
# 작업 규칙

> {{작업 원칙 한 줄 요약}}

---

## 규칙 등급

규칙은 강제성에 따라 분류된다:

- **[MUST]** — 강제 제약. 훅이나 구조적 검사로 강제됨. 명시적 승인 없이는 재정의 불가.
- **[SHOULD]** — 강력한 권장사항. 맥락이 요구할 때 재정의 가능하나, 이유를 문서화해야 함.

---

## 1. [MUST] 중간 산출물 위치

**에이전트 실행 중 생성되는 중간 산출물은 반드시 `_workspace/` 하위에 저장한다.**

- 파일명 컨벤션: `{phase}_{agent}_{artifact}.{ext}` (예: `01_planner_plan.md`, `02_writer_draft.md`)
- 오케스트레이터가 Phase 1 준비 단계에서 `_workspace/` 폴더를 직접 생성한다.
- 최종 산출물은 오케스트레이터가 `_workspace/` 내용을 취합하여 사용자 지정 경로에 생성한다.
- `_workspace/`는 감사 추적용으로 보존한다.

## 2. [MUST] 세션 관리 파일 사용

세션 관리 파일은 다음과 같은 역할을 한다. 각 파일의 용도를 명확히 구분하여 사용할 것.

- **`BRIEF.md`** — 이번 세션의 구체적 목표. ROADMAP의 한 조각으로, 이번 세션에서 완료할 수 있는 크기로 작성.
- **`HANDOFF.md`** — 세션 간 연결 문서. 이번 세션의 결과를 정리하고 다음 세션의 출발점을 제공.
- **`ROADMAP.md`** — 장기 계획. BRIEF들이 모여 이루는 큰 그림.

사용 시점:
- **세션 시작**: BRIEF.md 확인 → 이번 세션 목표 파악
- **세션 종료**: HANDOFF.md 업데이트 → 다음 세션을 위한 연결 고리 생성
- **방향 재검토**: ROADMAP.md 참조 → 장기 목표 대비 현재 위치 확인

## 3. [MUST] {{규칙 카테고리}}

{{설명}}

- **{{하위 규칙}}**: {{설명}}

---

## 3. [SHOULD] {{규칙 카테고리}}

{{설명}}

---

*최종 업데이트: {{YYYY-MM-DD}}*
```

> **harness v2 개선사항:** `_workspace/` 관리 규칙을 MUST 섹션 1번에 추가했습니다.

## BRIEF 섹션 (루트 BRIEF.md 용)

> 이번 세션에서 달성할 구체적이고 작은 목표. ROADMAP의 한 조각.

```markdown
# 세션 브리프

## 초점

{{이 세션이 무엇에 관한 것인지 한 줄 설명}}

> 이것은 ROADMAP의 단기 목표 중 하나로, 이번 세션에서 완료할 수 있는 크기여야 함.

## 목표

1. {{구체적이고 측정 가능한 목표}}
2. {{구체적이고 측정 가능한 목표}}
3. {{구체적이고 측정 가능한 목표}}

## 맥락

- {{관련 배경 정보}}
- {{관련 문서나 결정 사항 링크}}

## 제약사항

- {{기술적 또는 비즈니스 제약사항}}
- {{범위 경계}}

## 완료 조건

- [ ] {{기준 1 — 구체적이고 검증 가능}}
- [ ] {{기준 2 — 구체적이고 검증 가능}}
- [ ] {{기준 3 — 구체적이고 검증 가능}}

> **규칙**: 세션 종료 전에 모든 항목이 체크되어야 함. 미완료 시 명시적 사용자 재정의 필요.

---

*최종 업데이트: {{YYYY-MM-DD}}*
```

## ROADMAP 섹션 (루트 ROADMAP.md 용)

> 장기 계획 문서. BRIEF는 이 ROADMAP의 단기 목표 중 하나를 이번 세션에서 실행하는 것.

```markdown
# {{제목}}

## 맥락

이 변경이 왜 이루어지는지 — 해결하는 문제나 필요, 무엇이 촉발했는지, 의도된 결과.

> ROADMAP은 장기 방향을 제시하고, 각 세션의 BRIEF는 이 ROADMAP의 한 조각을 구체적으로 실행한다.

---

## 목표

1. 
2. 
3. 

## 비목표

1. 
2. 

## 제약사항

- 
- 

## 진행 상황

### 완료

- [x] 

### 대기 중

- [ ] 

## 검증

변경 사항을 종단 간 테스트하는 방법:
1. 
2. 

---

*계획 생성일: {{날짜}}*
```

## CLAUDE 섹션 (루트 CLAUDE.md 용)

```markdown
# {{워크스페이스|프로젝트}} 헌법

## 정체성

{{이 범위에서 Claude 인스턴스가 누구인지. 한 문장.}}

## 언어

- **{{사용자 대면 커뮤니케이션 언어}}**: 모든 사용자 대면 커뮤니케이션(채팅, 질문, 설명)
- **{{문서 작성 언어}}**: 모든 체크인 문서, 코드 주석, 파일 내용

## 핵심 지시사항

1. **{{지시사항 1}}** — {{설명}}
2. **{{지시사항 2}}** — {{설명}}
3. **{{지시사항 3}}** — {{설명}}

## 컨텍스트 참조 규칙

1. **세션 시작 시**: `context/ABOUT-ME.md` + `context/BRAND-VOICE.md` 읽기
2. **작업 수행 전**: `BRIEF.md` + `context/WORKING-RULES.md` 읽기
3. **도메인 용어 확인**: `context/GLOSSARY.md` 참조
4. **작업 완료 후**: `context/LESSONS.md`에 학습 사항 기록, `HANDOFF.md` 업데이트

## 금지사항

{{상위 규칙을 재정의하는 범위별 제약사항. 재정의가 없으면 생략.}}

---

*최종 업데이트: {{YYYY-MM-DD}}*
```

> **harness v2 개선사항:** 컨텍스트 참조 규칙 섹션을 추가했습니다. 에이전트/스킬이 자동으로 컨텍스트를 읽도록 유도합니다.
