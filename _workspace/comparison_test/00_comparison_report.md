## 정성 비교 — 핵심 차이

### 1. 추상 vs 구체 (역할 인트로)

**Run A**: "한국 푸드테크 스타트업의 신규 배달 앱 백엔드 개발을 담당하는 에이전트."

**Run B**: "서울 강남구에 거주하는 30대 초반 응용 소프트웨어 개발자. 결제 시스템 7년차로, 안정성과 장애 대응에 집중하며 SI 출신답게 문서화·보고 라인을 중시한다. 한 가정의 가장으로서 워라밸과 시스템 신뢰성을 동시에 챙긴다."

→ Run B는 **데이터셋의 demographics + professional_persona를 결합**하여 즉시 식별 가능한 캐릭터를 만든다.

### 2. 작업 원칙 — 가치 명제 vs 행동 패턴

**Run A**: "신뢰성 우선", "확장성 고려", "테스트 우선", "문서화" — 어느 백엔드 팀에도 통하는 원칙.

**Run B**: 같은 신뢰성 가치를 다음과 같이 페르소나화:
- "장애가 먼저, 기능은 그 다음" (결제 도메인 7년차의 우선순위)
- "보고 라인을 의식한 의사결정" (SI 출신 + 한국 직장 문화)
- "후배 멘토링" (7년차로서의 위치)
- "워라밸과 온콜의 균형" (가정 페르소나 반영, rota 명시화 요구)

### 3. Voice & Tone — Run A에는 없음

Run B에만 존재하는 1인칭 발화 샘플:
- "이번 스프린트에서 PG 연동 부분은 제가 정리해서 PR 올리겠습니다."
- "결제 정합성 이슈는 한 번 더 검토해 보면 좋을 것 같은데요."
- "온콜 로테이션은 다음 주 화요일에 한 번 정리해서 공유드릴게요."

이는 합쇼체+해요체 혼용, 우회적 이견 표현, 컨펌 톤이 자연스럽게 결합된 한국 IT 팀 발화. 단순 영어 직역으로는 만들 수 없는 톤.

### 4. 한국 직장 매너의 명시화

**Run B만 등장**:
- "단정하지 않고 ~로 진행해도 될까요? 식 컨펌 요청"
- "직접 반박보다 다른 측면도 한 번 고려해 보면 좋을 것 같습니다 우회 표현"
- "후배 PR 리뷰 시 격려 코멘트"
- "핫픽스는 팀장 컨펌 후 배포"

### 5. 금기 항목 — Run A는 빈 자리

Run B의 금기:
- "당신은 ~하세요" 같은 어색한 직역체
- 후배에게 갑작스러운 반말
- 영어 약어 남발 (AS, ETA, CTA를 한국어 맥락 없이)
- 결제 이슈를 가벼운 톤으로 ("그냥 ~하면 되죠")

이는 **다른 영어 베이스 에이전트가 만들기 쉬운 안티패턴**을 사전 차단.

## 결론

같은 프롬프트, 같은 역할에서 **두 스킬은 본질적으로 다른 산출물**을 만든다:

- **Run A** → "직무 명세서" (job description). 어떤 사람이 해야 할 일.
- **Run B** → "1인칭 페르소나 캐릭터". 누가, 어떤 톤으로, 어떤 매너로.

본 비교 테스트는 `korean-persona-harness`가 추가한 가치(=한국 페르소나·문화·언어 깊이)가 정량적·정성적으로 명백함을 보여준다. 단, Run B는 페이로드가 약 2.4배 — 페르소나가 불필요한 추상 작업(예: 인프라 설계)에서는 기존 `harness`가 적합. 둘은 **트리거 분기**로 사용하는 것이 옳다.
